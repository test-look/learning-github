# TASK.md — 变体 C：MVP 渐进迭代

## 策略声明

> **核心理念：能用 > 完美。先跑起来，再从真实反馈中迭代。30 分钟出一个可用版本，然后逐步长肉。**

本变体要求在 **30 分钟内**交付第一个可用的 Markdown 预览器（只支持最核心的 5 个语法），然后每 15 分钟迭代一个版本，逐步增加功能。你不是在设计完美系统，而是在进化一个活着的产品。

---

## 开发流程

### 迭代 0：极简 MVP（目标：30 分钟）

**只实现这 5 个语法**：
1. 段落（双换行分段）
2. 标题（`#` ~ `######`）
3. 粗体和斜体（`**` 和 `*`）
4. 行内代码（`` ` ``）
5. 无序列表（`- `）

**除此之外什么都不要做**。不要写测试，不要设计架构，不要考虑主题切换、代码高亮、表格、任务列表。就是一个能跑的编辑器。

**代码要求**：
- 单文件 `index.html`，CDN 引入 React + Tailwind + Babel
- 解析器可以简单粗暴——几个正则替换就行，不需要设计模式
- CSS 用 Tailwind，能用 utility class 就别写 CSS
- 完成度不需要 100%，但**必须能跑**

**验收标准（就这几条）**：
- [ ] 页面能打开
- [ ] 左侧输入 Markdown，右侧能看到 HTML 渲染
- [ ] 标题、粗体、斜体、代码、列表这 5 个语法正确渲染
- [ ] 没有控制台报错

**完成后立即截图**，记录 MVP 的状态。这是你的基准线。

---

### 迭代 1：补全核心语法（目标：15 分钟）

**新增功能**：
- 代码块（围栏代码块 `` ``` ``）
- 链接 `[text](url)` 和图片 `![alt](url)`
- 删除线 `~~text~~`
- 引用块 `> quote`
- 分割线 `---`

**重构机会**（如果你发现代码开始混乱）：
- 把解析逻辑抽成一个函数 `parseMarkdown(text)`
- 如果已经在函数里了，review 一下要不要拆分成更小的函数

**验收**：
- [ ] 上述 5 个新语法全部正确渲染
- [ ] 链接可以点击并新窗口打开
- [ ] 代码块有视觉区分（不同于行内代码）

---

### 迭代 2：列表与表格（目标：15 分钟）

**新增功能**：
- 有序列表
- 列表嵌套（缩进）
- 任务列表 `- [ ]` / `- [x]`
- GFM 表格

**性能注意**：随着语法规则增多，检查一下 5000 字文本是否有明显卡顿。

**验收**：
- [ ] 有序列表渲染正确
- [ ] 二级嵌套列表正确缩进
- [ ] 任务列表显示为 checkbox
- [ ] 表格正确渲染（表头 + 分隔行 + 数据行）

---

### 迭代 3：安全 + 进阶功能（目标：15 分钟）

**新增功能**：
- XSS 防护（`<script>` 转义）
- HTML 标签白名单放行（`<div>`, `<span>` 等安全标签）
- 自动链接（裸 URL → 可点击链接）
- 代码块复制按钮

**安全检查**：
- 输入 `<script>alert('xss')</script>` 不应弹窗
- 所有 `<a>` 有 `rel="noopener noreferrer" target="_blank"`

**验收**：
- [ ] XSS 注入不弹窗
- [ ] 安全 HTML 标签（如 `<kbd>Ctrl</kbd>`）能正确渲染
- [ ] 代码块复制按钮可用

---

### 迭代 4：UI 打磨（目标：20 分钟）

**新增功能**：
- 亮色/暗色主题切换
- 编辑区行号显示

- 复制 HTML 源码按钮
- 响应式布局（移动端上下布局）

**只用 CSS / Tailwind 解决响应式，不要引入 JS 逻辑判断屏幕宽度**。

**验收**：
- [ ] 全部 P0、P1、P2 功能正常
- [ ] 移动端布局正确切换
- [ ] 主题切换平滑无闪烁
- [ ] 状态栏实时更新

## 代码规范

### MVP 阶段的「可以暂时不遵守」
- ❌ 完美命名（`func1`、`x` 这种不行，但 `parseMd`、`doBold` 这类可以）
- ❌ 完整注释（代码自己说清楚就行，不需要 JSDoc）
- ❌ 测试（等 MVP 跑通后视情况补）
- ❌ 性能优化（等感觉到卡顿了再优化）

### 所有阶段都必须遵守的
- ✅ 使用 const / let（不用 var）
- ✅ 解析器用纯函数（输入 Markdown，输出 HTML 字符串）
- ✅ 所有 HTML 输出在渲染前做 XSS 转义
- ✅ 单文件 `index.html`，CDN 依赖

### MVP 后的代码整理
每次迭代结束时花 2 分钟：
1. 有没有重复的代码可以提取？
2. 函数名是否还准确反映它的职责？
3. 有没有应该用 `useMemo` 但没用重计算？

基于 MVP 渐进策略，这个版本的产物预期特点是：

- 🟡 边界情况可能有遗漏（没有测试强制覆盖）
