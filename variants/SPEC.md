# 需求规格说明 — Markdown 实时预览编辑器

## 1. 项目概述

### 1.1 产品名称

MarkView — Markdown 实时预览编辑器

### 1.2 一句话描述

一个单页面 Web 应用，左侧输入 Markdown 文本，右侧实时渲染为 HTML，所见即所得。

### 1.3 目标用户

开发者、技术写作者、需要快速将 Markdown 笔记转为格式化内容的任何人。

---

## 2. 技术栈

### 2.1 硬性约束

- **React 18**：通过 CDN 引入 `react@18` 和 `react-dom@18` 的 UMD 构建版本
- **Tailwind CSS 3**：通过 CDN 引入（Play CDN）
- **Babel Standalone**：通过 CDN 引入，用于在浏览器中编译 JSX
- **纯前端**：所有代码运行在浏览器中，不依赖 Node.js、npm、webpack 等
- **单 HTML 文件**：整个应用放在一个 `index.html` 文件中

### 2.2 推荐的 CDN 引入方式

```html
<!-- React 18 -->
<script crossorigin src="https://unpkg.com/react@18/umd/react.production.min.js"></script>
<script crossorigin src="https://unpkg.com/react-dom@18/umd/react-dom.production.min.js"></script>

<!-- Babel Standalone (for JSX) -->
<script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>

<!-- Tailwind CSS Play CDN -->
<script src="https://cdn.tailwindcss.com"></script>
```

### 2.3 浏览器兼容

- Chrome 100+, Firefox 100+, Safari 16+, Edge 100+
- 不需要支持 IE

## 3. 功能需求

### 3.1 核心功能：Markdown 解析与渲染（P0）

| ID | 功能 | 描述 |
|----|------|------|
| F1 | 实时预览 | 用户在左侧编辑区输入 Markdown，右侧实时更新 HTML 渲染结果。延迟控制在 100ms 以内。 |
| F2 | 标题 | 支持 H1-H6（`#` ~ `######`），正确映射到 `<h1>` ~ `<h6>` |
| F3 | 粗体/斜体/删除线 | `**粗体**` → `<strong>`，`*斜体*` → `<em>`，`~~删除线~~` → `<del>` |
| F4 | 行内代码 | `` `code` `` → `<code>`，带有视觉区分的背景 |
| F5 | 代码块 | 三个反引号包裹的代码块，支持语言标注（如 ` ```javascript `），语法高亮**
| F6 | 无序列表 | `- item` / `* item` → `<ul><li>` |
| F7 | 有序列表 | `1. item` → `<ol><li>` |
| F8 | 列表嵌套 | 子列表通过缩进表示（2 或 4 空格），支持有序/无序混用 |
| F9 | 引用块 | `> quote` → `<blockquote>`，支持嵌套 |
| F10 | 链接 | `[text](url)` → `<a href="url">text</a>` |
| F11 | 图片 | `![alt](url)` → `<img src="url" alt="alt">` |
| F12 | 分割线 | `---` / `***` / `___` → `<hr>` |
| F13 | 表格 | GFM 表格语法（管道符 + 连字符分隔表头） |
| F14 | 任务列表 | `- [ ]` 和 `- [x]` → 可勾选的 checkbox |

### 3.2 进阶功能（P1）

| ID | 功能 | 描述 |
|----|------|------|
| F15 | HTML 渲染 | Markdown 中混入的 HTML 标签应被安全渲染（但禁止执行 `<script>`） |
| F16 | 转义字符 | 反斜杠转义（如 `\*` 不渲染为斜体） |
| F17 | 自动链接 | 裸 URL 自动转为可点击链接 |
| F18 | 代码块复制按钮 | 每个代码块右上角有一个「复制」按钮，点击复制全部代码 |

### 3.3 体验功能（P2）

| ID | 功能 | 描述 |
|----|------|------|
| F19 | 编辑区行号 | 左侧编辑区显示行号 |
| F20 | 同步滚动 | 左侧滚动时右侧大致同步（尽力而为，不需要精确对齐） |
| F21 | 主题切换 | 支持亮色/暗色两种主题 |
| F22 | 字数/行数统计 | 底部状态栏显示字符数、行数 |
| F23 | 导出 HTML | 一键复制渲染后的 HTML 源码到剪贴板 |

## 4. 非功能需求

### 4.1 性能

- **首屏渲染**：页面加载后 1 秒内可交互
- **实时预览延迟**：输入到渲染更新不超过 100ms
- **大文本处理**：10000 字符输入时预览不应有明显卡顿

### 4.2 安全

- **XSS 防护**：Markdown 渲染为 HTML 时必须转义内嵌的 `<script>` 标签
- **链接安全**：外部链接添加 `rel="noopener noreferrer"` 和 `target="_blank"`

### 4.3 可用性

- **响应式设计**：桌面端左右分栏，移动端（宽度 < 768px）切换为上下布局
- **键盘友好**：Tab 键在编辑区正常插入制表符（不要失焦）

### 4.4 可维护性

- Markdown 解析逻辑应与 UI 组件分离
- 新增一个语法规则不应需要修改 UI 组件

## 5. 验收标准

### 5.1 功能验收

运行以下 Markdown 样本，右侧渲染结果应与 GitHub/GitLab 的渲染效果一致：

```markdown
# MarkView 测试文档

## 文本格式

这是**粗体**，这是*斜体*，这是~~删除线~~，这是`行内代码`。

## 列表

### 无序列表
- 项目一
- 项目二
  - 子项目 A
  - 子项目 B

### 有序列表
1. 第一步
2. 第二步
3. 第三步

### 任务列表
- [x] 已完成的任务
- [ ] 待完成的任务

## 引用

> 这是一段引用
> > 这是嵌套引用

## 代码块

\`\`\`javascript
function greet(name) {
    return `Hello, ${name}!`;
}
console.log(greet('World'));
\`\`\`

## 表格

| 姓名 | 年龄 | 城市 |
|------|------|------|
| 张三 | 28   | 北京 |
| 李四 | 32   | 上海 |

## 链接与图片

[GitHub](https://github.com)

https://www.google.com

## 分割线

---

**结束**
```

### 5.2 行为验收

1. 打开页面，左右两栏可见
2. 在左侧输入文本，右侧即时更新（无明显延迟）
3. 复制按钮工作正常
4. 暗色主题切换后页面不闪烁
5. 缩小浏览器到手机宽度，布局变为上下
6. 输入包含 `<script>alert('xss')</script>` 的文本，不应弹窗
7. 10000 字文本输入后，预览不卡顿

## 6. UX 参考

### 6.1 布局（桌面端）

```
┌──────────────────────────────────────────────────────┐
│  MarkView                           [🌙 暗色模式]    │
├──────────────────────┬───────────────────────────────┤
│                      │                               │
│   编辑区 (50%)       │   预览区 (50%)                │
│   textarea 风格     │   渲染后的 HTML               │
│   等宽字体           │   系统字体                      │
│   带行号             │                                │
│                      │                               │
├──────────────────────┴───────────────────────────────┤
│  📝 1234 字  │  📄 56 行  │  📋 复制 HTML             │
└──────────────────────────────────────────────────────┘
```

### 6.2 视觉风格

- 编辑区：深色背景 + 浅色文字（暗色模式）/ 浅色背景 + 深色文字（亮色模式）
- 预览区：GitHub 风格的 Markdown 渲染
- 分割线：可拖拽调整左右比例（加分项）
- 整体风格：专业、简洁，类似 VS Code 的 Markdown 预览
